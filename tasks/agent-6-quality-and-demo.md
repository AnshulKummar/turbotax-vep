## Tasks

### T-601: Cross-slice integration tests

`tests/integration/end-to-end.test.ts` walks the full flow:

1. Load the Mitchell return + TY2024 prior year (Agent 1)
2. Capture customer goals via the goal intake (Agent 2)
3. Call the pre-work API (Agent 3) and assert complexity = 8, at least 5 YoY deltas, at least 5 risk register entries
4. Call the recommendations API (Agent 2) and assert every `must_appear: true` recommendation from `src/data/golden-recommendations.json` is present
5. Simulate an expert action (accept the RSU double count recommendation) via the audit capture API (Agent 5)
6. Query the audit trail and assert the action landed
7. Call the "what AI saw" API and assert no raw PII leaked

Additional integration tests:

- Changing the goal mix changes the recommendation ranking
- Wash sale lot edit triggers the live quality co-pilot warning
- The expert minutes counter ticks against the right baselines

### T-602: Demo walk-through script

`docs/demo-script.md` is the step-by-step the developer follows during the live interview demo. It maps to the 10 demo points in `docs/PRD.md` Section 7. Each step includes:

- The exact click path
- The expected visible result
- A two-sentence narrator note explaining the product principle being demonstrated
- An estimated time (the whole demo should fit in 8 minutes)

### T-603: 50-return synthetic test set

`tests/calibration/test-set/return-001.json` through `return-050.json`. Generated by parameter sweep over:

- Filing status (single, MFJ, MFS, HoH)
- Income tier (under $50K, $50-150K, $150-300K, $300K+)
- W-2 only vs W-2 + 1099 vs full schedule C
- Single state vs multi state (3 state combinations)
- Presence of: rental, K-1, RSU, HSA, wash sale, ISO exercise, foreign income
- Prior year continuity (yes/no)

For each return, a `ground_truth.json` sibling lists the rule findings the engine should detect, so Agent 5's calibration harness can score predictions.

### T-604: Success metric dashboard route

`app/(workbench)/metrics/page.tsx` renders the Section 8 success metrics live from the audit trail and the calibration_runs table:

- Goal dashboard: % of recommendations carrying a goal fit score
- Recommendation list completeness: 5/5 mechanically detectable errors caught (status badge)
- Expert minutes per return on the Mitchell synthetic
- Confidence calibration: latest calibration_runs row, max calibration error, pass/fail
- PII redaction: latest test set result, leakage count
- Routing rationale chip: dimensions covered (4/4)
- Audit trail: 100% capture rate
- Cases auto-closed without warning: count

### T-605: CI gate

`.github/workflows/ci.yml` runs:
- `npm run lint`
- `npm test`
- `npm run calibration` (the calibration eval harness; blocks merge on max error > 5pp)
- `npm run build`
